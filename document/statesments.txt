업체를 등록한다.

WIFI기기를 등록한다.

LTE기기를 등록한다.

용접기를 등록한다.

프로젝트를 등록한다.

거래내역을 등록한다.

1. 계약을 하면 관련 업체를 등록한다.
2. WIFI기기를 구입하면 해당 기기를 등록한다.
3. LET기기를 구입하면 해당 기기를 등록한다.
4. 용접기를 만들면 해당 기기를 등록한다.
5. 계약과 관련된 프로젝트를 등록한다. (설치장소등의 내용을 등록한다.)
6. 하나의 납품 계약을 하면 하나의 프로젝트가 생성한다.
7. 프로젝트에 관련된 WIFI기기/LTE기기/용접기/용역내용을 등록하고 수정할 수 있다.
8. 해당 건들을 요약해서 보여주는 부분이 있다. (거래 회차 별로 따로 기록 할 수 있어야 한다. 즉 거래 회차가 자동으로 생성될 수 있어야 한다. (만일 이전 거래회차 흠....)
9. 기기는 이력이 남아야 한다.


7. 프로젝트내에 거래내역이 존재한다. 이 것은 용접기
   WIFI / LTE기기의 종류이다.

시나리오1. 
1.   A업체와 계약을 한다.
2.   A업체는 WIFI기기 1대와 CO2용접기 3대를 구입하기로 계약하였다. 2026/1/1 계약하였고 2026/1/31일 납품하기로 약정하였다.
3.   이중 1대의 기기에는 경광등 설치를 추가했다. 
4.   A업체에 추가적인 네트워크 설치의 용역을 수행하기로 계약을 추가하였다.
5.   이후 A업체는 CO2용접기 1대를 추가로 구입하기로 계약하였다.
6.   이후 A업체는 용접기 1대를 불량의 이후로 교체를 요구하여 해당 기기를 교체하였다.
7.   6개월후 WIFI기기가 이상하여 해당 기기를 교체하였다.
   -> 이때 세금계산서를 어떻게 발행하는가?
8.   1년이 지난후 CO2용접기 3대를 추가로 발주하여 구매하기로 하였다. 이 계약은 10%할인하여 공급하기로 하였다.


0. 우성기전 업체 등록
1. A업체 정보 등록
   아이디 / 이름 / 전화번호 / 주소 / 담당자
2. WIFI기기 등록
   아이디 / 모델명 ...
3. CO2용접기 3대 등록
   아이디 / 모델명 / 추가사항 등
4. A업체 공급. 프로젝트 ProjA를 등록한다.
   프로젝트 : 아이디 / 업체정보 / 설치업체정보 / 비고
5. PA에 용역계약을 추가한다.
6. CO2용접기 한대에 경광등 설치 부가옵션을 추가한다.
7. ProjA에 용역계약을 추가한다.
8. ProJA에 WIFI기기를 추가한다.
9. ProJA에 용접기 1, 2, 3을 추가한다.
10. 이 내역을 하나의 거래내역으로 저장한다. 이 거래내역을 Activity1이라고 한다.
   거래내역 : 아이디 / 프로젝트정보 / 설치위치 / 비고 / 계약일 / 납품일
   집행정보 : 용역 / WIFI / LTE / 용접기
              판매단가 / 판매금액 / 비고 
              사용단가 / 사용시작일 / 사용종료일 / 사용금액 / 사용금액 / 상태
11. Activity1에 CO2용접기 1개를 추가한다.
12. Activity1에서 CO2용접기 1대를 제거한다.
13. Activity1에 CO2용점기 1대를 추가한다.
14. Activity1에 CO2용접기 1대를 제거하였다.
    해당 입력 / 삭제 / 수정 이력이 남아야 한다.
15. 1년후 별도의 거래이므로 거래내역을 새롭게 생성하는 것으로 한다.
16. Activity2에 용접기 4, 5, 6를 추가한다.
17. 이 내역을 하나의 거래내역으로 저장한다. 이 거래내역을 Activity2라고 한다.
